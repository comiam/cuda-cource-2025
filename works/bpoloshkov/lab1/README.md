# Lab 1: CUDA Graphics

## Описание

Программа рисует круг в терминале с использованием параллельных вычислений на GPU. Каждый пиксель ASCII-изображения вычисляется отдельным потоком CUDA.

## Требования

- CUDA Toolkit 11.0+
- GPU с поддержкой CUDA
- Linux / macOS с nvcc
- Make

## Компиляция

```bash
make
```

## Запуск

```bash
./graphics [size]
```

Параметр `size` — размер фигуры от 10 до 100 (по умолчанию 20).

### Примеры

```bash
./graphics
./graphics 30
./graphics 50
```

```

  ***           *** 
 ***             ***
 ***             ***
 ***             ***
  ***           ***
```

## Алгоритм

Для каждого пикселя (x, y) на холсте вычисляется расстояние до центра круга:

```
dist = sqrt(dx² + dy²)
```

где:
- `dx = x - cx` — горизонтальное расстояние от центра
- `dy = (y - cy) * 2` — вертикальное расстояние с коррекцией пропорций

Символы в терминале примерно в 2 раза выше, чем шире, поэтому `dy` умножается на 2 для получения визуально круглой формы.

Если расстояние близко к радиусу (`|dist - radius| <= 1.5`), пиксель отмечается как часть окружности.

## Используемые функции

| Функция | Описание |
|---------|----------|
| `cudaMalloc` | Выделяет память на GPU |
| `cudaMemset` | Заполняет память на GPU заданным значением |
| `cudaMemcpy` | Копирует данные между CPU и GPU |
| `cudaDeviceSynchronize` | Блокирует CPU до завершения всех операций на GPU |
| `cudaFree` | Освобождает память на GPU |
| `sqrtf` | Вычисляет квадратный корень (float) |
| `fabsf` | Возвращает абсолютное значение числа (float) |

## Особенности реализации

- 2D сетка потоков: каждый поток вычисляет один пиксель
- Блоки размером 16×16 потоков
- Макрос `CUDA_CHECK` для проверки ошибок CUDA-вызовов
